Role Boundaries

  • Admin reviews system-wide data but does not replace Homeowner, Developer, or Legal workflows.
  • Homeowner accounts submit defects, update descriptions, upload evidence, and add remarks.
  • Developer accounts update defect progress, completion dates, and response status.
  • Legal accounts generate tribunal-focused reports using the verified case records.

Defect Monitoring

  • Use the Defects tab to review all recorded defects across units and projects.
  • Check pending, delayed, overdue, completed, and closed totals before report generation.
  • Evidence count helps identify defects that may be incomplete for formal reporting.
  • Project mapping uses unit numbers, with unmapped records grouped as Others / Unrelated.

Audit And Traceability

  • Audit Log stores database-backed activity such as login events, status updates, and evidence uploads.
  • Delete-defect records are stored separately in audit_data/delete_defect.log.
  • Use filters to narrow audit records by role, action, or date.
  • Recent login time is read from the latest successful login audit event.

Reports And Backups

  • Version History tracks generated report versions by role and language.
  • Backup Status shows the latest stored snapshot under audit_data/backups.
  • Review backup age before major demonstrations, submissions, or data cleanup.
  • Keep report generation scoped to the correct role account for accurate case context.
